# The 2nd Summit - Project Guidelines

This is the website for The 2nd Summit (the2ndsummit.com).

## Reference Docs

Consult these before starting work in their area. Do not duplicate their contents here.

- `BRAND.md`: brand plan, voice, audience, and content pillars. Consult before writing any site copy, metadata, or content.
- `DESIGN.md`: the "Contour" design system (palette, typography, components, pointer-site rules). Consult before writing or changing any HTML/CSS.

## Writing Style Rules

These rules apply to ALL text written in this project: site copy, metadata, markdown docs, comments, commit messages, and generated content.

- NEVER use em dashes (—) or en dashes (–). Use a comma, colon, period, or plain hyphen (-) instead.
- NEVER use emoticons or emoji (no :), ;-), or Unicode emoji characters).

## Brand Rules

- The brand name is always written as "The 2nd Summit", never "The Second Summit". Lowercase prose references to the concept of a second summit are fine.
- Author attribution is always "Hal Hunt" (site metadata, bylines, channel credits).
- Canonical URL: https://the2ndsummit.com
